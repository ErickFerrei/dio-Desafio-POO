# ☕ Desafio POO com Java — Desmistificando a Orientação a Objetos
 
> Pratique os quatro pilares da Programação Orientada a Objetos com um problema real e evolua sua capacidade de abstração usando Java.
 
---
 
## 📖 Sobre o Desafio
 
Este desafio tem como objetivo **desmistificar a Programação Orientada a Objetos (POO)** por meio de uma implementação prática em Java. Aqui você vai além da teoria e coloca a mão na massa com um problema do mundo real, aplicando os conceitos fundamentais que todo desenvolvedor Java precisa dominar.
 
A ideia é simples: entender como modelar o mundo real em código de forma organizada, reutilizável e elegante.
 
---
 
## 🏛️ Os Quatro Pilares da POO
 
### 🔷 1. Abstração
> *"Modelar apenas o que é essencial."*
 
Identificar as características e comportamentos relevantes de um objeto do mundo real e representá-los em uma classe, ignorando detalhes desnecessários para o contexto.
 
### 🔐 2. Encapsulamento
> *"Proteger e controlar o acesso aos dados."*
 
Ocultar os detalhes internos de implementação e expor apenas o necessário por meio de métodos públicos (`getters` e `setters`), garantindo a integridade do estado do objeto.
 
### 🧬 3. Herança
> *"Reutilizar e especializar comportamentos."*
 
Permitir que uma classe filha herde atributos e métodos de uma classe pai, promovendo o reúso de código e a criação de hierarquias lógicas entre entidades.
 
### 🎭 4. Polimorfismo
> *"Um mesmo comportamento, múltiplas formas."*
 
Capacidade de um objeto ser tratado como uma instância de sua classe pai, enquanto executa o comportamento definido em sua própria classe — via sobrescrita (`@Override`) ou sobrecarga de métodos.
